## 1. Что это за продукт

**Каллисто** — веб-платформа для управления бытом в общежитии:

- расписание дежурств;
- чек-листы уборки;
- учёт выполнения;
- рейтинги;
- связь жильцов с администрацией.

**Целевая среда:** корпуса **C1, C2, R1–R6**, квартиры до **8 жильцов**, роли **житель → ответственный (manager) → администратор**.

**Доступ:** браузер, без установки приложения. Задел: **Telegram-бот**, **WebSocket-уведомления**.

---

## 2. Архитектура ролей

```
Гость / новый резидент
    → лендинг → регистрация → выбор квартиры

Жилец
    → расписание, чек-лист, обмен днями, профиль, рейтинг
    → /dashboard

Ответственный за квартиру (manager)
    → всё жильца + настройка задач, шаблоны, состав жильцов

Администрация Каллисто
    → жители, квартиры, проверки, мероприятия, санкции, доп. задачи
    → /admin (квартиру не выбирает)
```

**Важно:** админы не «заселяются» в квартиру — управление кампусом отделено от быта жильцов.

Права администратора: логин в **`ADMIN_USERNAMES`** (`.env`, например `Admin,Admin1`) + флаг **`is_admin`** в БД.

## 3. Функционал по ролям

### 3.1. Обычный пользователь — жилец

**Вход:** `/dashboard`. Админ-панель `/admin` недоступна.

#### Вкладка «Уборка»

| Функция | Описание |
|--------|----------|
| Расписание на неделю | 7 дней (Пн–Вс), кто занял день, подсветка «сегодня» |
| Занять / освободить день | Закрепить или снять слот в расписании квартиры |
| Обмен днями | Запрос соседу, принять / отклонить входящие |
| Чек-лист задач | Список по дню, прогресс в % |
| Отметить задачу | Только в **свой** день уборки по расписанию |
| Своя задача на день | Добавить кастомную задачу для всей квартиры |
| Удалить свою задачу | Только созданные самим |
| Лимит уборок | До N в неделю (по умолчанию 2); засчёт при полном чек-листе |
| Фото уборки | Подтверждение за день |
| Напоминания | Баннер: кому и когда убирать |
| История уборок | Кто и когда убирал (админы в истории не показываются) |
| Комментарии к задачам | Обсуждение задачи в чек-листе |

#### Вкладка «Квартира»

| Функция | Описание |
|--------|----------|
| Выбор / смена квартиры | Заселение, переселение, «Покинуть» |
| Поиск квартир | По корпусу и номеру |
| Карточка квартиры | Корпус, номер, жильцы, счётчик уборок |
| Жильцы | Соседи, роли, переход в профиль |
| Описание и фото квартиры | Текст «о нас», аватар квартиры |
| Проверки | Просмотр проверок и нарушений (только чтение) |
| Публичная страница | `/apartments/{id}` для шаринга |

*Настройка типа уборки и шаблонов — только у ответственного.*

#### Вкладка «Рейтинг»

- Лидерборд **жильцов** и **квартир** (фильтр по корпусу и периоду).
- Администраторы в рейтинге **не участвуют**.

#### Вкладка «Администрация»

- Список администраторов Каллисто (ФИО, аватар, профиль).

#### Профиль, уведомления, аккаунт

| Область | Возможности |
|--------|-------------|
| Профиль `/profile` | ФИО, био, аватар, рамки (геймификация), смена пароля, задачи «мой день уборки» |
| Колокольчик | Мероприятия (прочитано), доп. задания от админа, WebSocket в реальном времени |
| Аккаунт | Регистрация, вход (логин или email), сброс пароля, просмотр `/profile/{id}` |

**Ограничения жильца:** не отмечает задачи чужого дня; не исключает соседей.

---

### 3.2. Обычный пользователь — ответственный (manager)

Все возможности **жильца**, плюс:

| Функция | Описание |
|--------|----------|
| Настройка задач | Лёгкая / генеральная уборка или свои шаблоны (под «Жильцы») |
| Шаблоны | Создать / удалить шаблоны чек-листа |
| Исключить жильца | Выселить соседа (себя — через «Покинуть») |
| Рамка квартиры | В профиле — за достижения квартиры |

Первый в пустой квартире автоматически становится **ответственным**.

---

### 3.3. Администратор

**Вход:** редирект на `/admin`. С `/dashboard` — обратно в админку. **Квартиру не выбирает.**

#### Шапка

| Функция | Описание |
|--------|----------|
| Профиль | ФИО, аватар, **цвет обводки** админа |
| CSV уборок | Выгрузка истории по кампусу |
| Выход | Logout |

#### Вкладка «Жители Каллисто»

- Поиск по ФИО, логину, email.
- **Администраторы** (отдельный блок) · **Жители** (корпус-квартира, уборки) · **Заблокированные**.
- Действия: **Управление**, **Профиль** (`/users/{id}/admin` для расширенных данных).

**Модальное «Управление» (житель):**

- квартира;
- **рейтинг** (число уборок вручную);
- **доп. задания** на неделю (назначить / удалить);
- **заблокировать** (не себя и не другого админа).

#### Вкладка «Квартиры»

| Функция | Описание |
|--------|----------|
| Выбор квартиры | Фильтр по корпусам C1, C2, R1–R6… |
| Жильцы | Список, роли |
| Сделать ответственным | Назначить manager |
| Исключить | Выселить жильца |
| Проверки | Создать / удалить inspection |
| Нарушения | Категория, баллы, комментарий; удаление |

Жильцы видят проверки во вкладке «Квартира» → «Проверки».

#### Вкладка «Мероприятия»

- Создание: название, описание, дата/время, картинка.
- Публикация → уведомление всем (WebSocket + колокольчик).
- Список: просмотр, удаление.

#### Профиль администратора

- Настройки, смена пароля, цвет обводки.
- **Нет** расписания, чек-листа, истории уборок, доп. заданий как у жильца, рейтинга.

**Админ не делает:** заселение, дни в расписании, отметки уборки, участие в лидерборде.

## 5. Что автоматизирует

### Для жильцов

| Было вручную | Делает Каллисто |
|--------------|-----------------|
| График в чате / на бумаге | Недельное расписание: занять / освободить слот |
| Споры «кто убирал» | Чек-лист, прогресс %, автозасчёт уборки |
| Забыли дежурство | Напоминания + WebSocket |
| Обмен в личке | Запросы на обмен дней |
| Нет доказательств | Фото + история уборок |
| Непонятен лидер | Рейтинг жильцов и квартир |
| Поручения админа | Доп. задания с уведомлением |
| Учётные записи | Регистрация, вход, сброс пароля |
| Поиск соседей | Поиск квартир, публичная страница |

### Для ответственного

- Тип уборки (лёгкая / генеральная) и свои шаблоны.
- Состав жильцов, описание и аватар квартиры.
- Просмотр проверок по квартире.

### Для администрации

- Реестр жителей (админы / жители / заблокированные).
- Квартиры по корпусам, ответственные, выселение.
- Проверки и нарушения (категория, баллы, комментарии).
- Доп. задания, корректировка рейтинга, блокировка.
- Мероприятия с push-уведомлениями.
- Экспорт CSV истории уборок.
- Задел: роли по корпусам (диспетчер / наблюдатель).

### Техническая автоматизация (для IT)

- Автосоздание слотов расписания на неделю.
- Автогенерация стандартных задач под режим уборки.
- Засчёт уборки только при 100% чек-листе и лимите в неделю.
- Синхронизация админов (`ADMIN_USERNAMES` + БД).
- CI: pytest + сборка frontend.

## 6. Какие процессы ускоряет

1. **Согласование графика** — минуты вместо переписки на 1–2 дня.
2. **Контроль выполнения** — пропуски, история, рейтинг без обхода «на словах».
3. **Нарушения при проверках** — в системе, не в Excel/бумаге.
4. **Массовые коммуникации** — мероприятия всем сразу.
5. **Онбординг** — регистрация → квартира → расписание и чек-лист.
6. **Отчётность** — CSV для руководства и аудита.
7. **Мотивация** — рамки за рейтинг и уборки.

**Эффект:** на корпус со сотнями жильцов сводки и разборки сжимаются до **поиска в панели + автоматических отчётов** — меньше нагрузки на персонал и конфликтов.

## 7. Почему необходимо компании

### Риски без системы

- Конфликты из‑за непрозрачного графика — репутация кампуса.
- Нет единого источника правды (чаты, Excel, сторож).
- Масштаб: десятки корпусов, сотни квартир — ручной учёт не тянет.
- Сложно привязать нарушения ПХД/санитарии к людям и датам.
- Падает дисциплина без мотивации и прозрачности.

### Что даёт внедрение

| Выгода | В продукте |
|--------|------------|
| Прозрачность | Расписание, история, рейтинг, фото |
| Управляемость | Админ-панель, проверки, блокировки, доп. задачи |
| Масштабируемость | Корпуса C/R, единая БД |
| Цифровой след | История, нарушения, CSV |
| Коммуникация | Объявления + WebSocket |
| Меньше ручной работы | Данные в системе, не в чатах |
| Имидж | Цифровой кампус вместо хаоса в мессенджерах |

Для **образовательного / промышленного кампуса (Алабуга и аналоги)** — стандартизация быта резидентов и **data-driven** управление общежитием.

## 8. Дорожная карта

### Этап 1 — Стабильная эксплуатация (0–3 мес.)

- Продакшен: PostgreSQL, деплой, Sentry.
- Регламент `ADMIN_USERNAMES`, обучение диспетчеров.
- Telegram-бот с напоминаниями (каркас есть).

### Этап 2 — Углубление контроля (3–6 мес.)

- PWA / push.
- Дашборды руководства: пропуски, нарушения, динамика.
- Интеграция HR / заселение.
- Автоназначение дней по ротации.
- Штрафы / баллы дисциплины.

### Этап 3 — Экосистема кампуса (6–12 мес.)

- SSO.
- Ремонт, бронь помещений, опросы.
- AI: риск пропусков, рекомендации графика.
- RU/EN, white-label для других общежитий.

### Этап 4 — B2B / SaaS

- SaaS для общежитий и коливингов.
- Тарифы: базовый / pro.
- API и отчёты для собственников.

**Vision:** *«Каллисто — операционная система быта кампуса: от дежурств в одной комнате до аналитики дисциплины по всем корпусам».*

## 10. Питч на 1 минуту

> В общежитии сотни людей делят уборку на словах — отсюда конфликты и претензии к администрации. **Каллисто** переводит это в цифру: недельное расписание, чек-лист с автозасчётом уборки, история и рейтинг, обмен днями, фото. Жильцы работают в браузере за минуты; ответственный настраивает задачи; администрация видит жильцов, квартиры, проверки, нарушения, мероприятия и выгружает отчёты по корпусам C и R.
>
> Сейчас это рабочий MVP с тестами и админкой. Дальше — аналитика, Telegram, интеграции с заселением, выход на другие кампусы. Для компании это не «ещё один чат», а **управляемый стандарт быта** и снижение операционных рисков.
